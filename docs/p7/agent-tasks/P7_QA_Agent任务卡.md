# P7 QA Agent 任务卡 v1.0

## 当前任务

验证 **P7 活动规则页 / 规则说明页** 是否符合设计稿和业务口径。

## 测试范围

1. 页面视觉还原。
2. 从 P6 活动规则入口进入 P7。
3. 返回按钮行为。
4. 活动规则文案正确性。
5. 企微二维码展示。
6. 异常兜底。
7. 埋点触发。
8. 页面适配。

## 功能用例

| 用例 | 操作 | 预期结果 |
|---|---|---|
| 进入 P7 | 从 P6 点击「活动规则」 | 正常进入 P7 活动规则页 |
| 页面标题 | 打开 P7 | 标题展示「活动规则」 |
| 页面副标题 | 打开 P7 | 副标题展示「一举高中 · 六月牛气加油签」 |
| 返回上一页 | 从 P6 进入 P7 后点击返回 | 返回 P6 |
| 无历史返回 | 直接打开 P7 后点击返回 | 跳转 P1 首页 |
| 规则数量 | 查看规则卡 | 展示 1-7 共 7 条规则 |
| 规则 1 | 查看规则 1 | 文案为「每人每日默认1次抽签机会；」 |
| 规则 2 | 查看规则 2 | 文案为「分享好友参与并完成抽签，原用户增加1次助力进度；」 |
| 规则 3 | 查看规则 3 | 文案为「每日通过分享最多获得3次额外机会；」 |
| 规则 4 | 查看规则 4 | 文案为「每日完成抽签后，自动点亮1天进度，每天最多点亮1天；」，不能出现必须领取奖励后才点亮 |
| 规则 5 | 查看规则 5 | 文案体现累计点亮 7 天或邀请 5 位好友达标 |
| 规则 6 | 查看规则 6 | 文案说明奖品数量有限，以活动页面公示为准 |
| 规则 7 | 查看规则 7 | 文案说明活动不代表成绩预测 |
| 企微二维码 | 滚动到底部 | 正常展示二维码和「扫码添加企微」说明 |
| 二维码异常 | 模拟二维码加载失败 | 展示兜底提示，不导致页面崩溃 |
| 规则接口异常 | 模拟接口失败 | 展示本地兜底规则，不出现空白页 |
| 页面边界 | 小屏设备查看 | 页面可滚动，文字不被裁切 |
| 不应出现内容 | 查看 P7 全页 | 不出现抽签按钮、奖励卡、商品推荐卡、再抽一次按钮、签文结果、AI 解签正文 |

## 埋点验证

1. 进入页面触发 rules_page_view。
2. 规则渲染成功触发 rules_page_render_success。
3. 点击返回触发 rules_back_click。
4. 二维码曝光触发 rules_qrcode_exposure。
5. 滚动到底部触发 rules_scroll_to_bottom。
6. 模拟二维码加载失败触发 rules_qrcode_load_fail。
7. 模拟规则加载失败触发 rules_page_render_fail。

## 视觉检查

1. 深红背景与活动整体风格一致。
2. 返回按钮位置和点击热区合理。
3. 顶部标题、金色字体、牛肉礼盒氛围图与设计稿一致。
4. 规则卡边框、圆形序号、分割线、文字间距与设计稿一致。
5. 底部企微二维码模块不被安全区遮挡。

## 交付结果

1. 测试通过 / 不通过结论。
2. 缺陷列表。
3. 埋点验证结果。
4. 适配设备说明。
5. 是否存在待产品确认的问题。
