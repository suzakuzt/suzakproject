# P7 后端 Agent 任务卡 v1.0

## 当前任务

为 **P7 活动规则页** 提供规则配置、企微二维码配置和埋点基础字段。

## 页面定位

P7 主要是规则展示页，业务上不直接修改用户分享进度、点亮进度或奖励状态。

## 需要提供 / 确认的能力

1. 提供活动规则配置接口，或明确当前阶段由前端静态兜底。
2. 返回页面标题、副标题、规则列表。
3. 返回企微二维码图片地址、二维码 ID、企微说明文案。
4. 返回 activity_id、session_id、source_page 等埋点所需字段。
5. 支持后续运营替换二维码，不需要前端重新发版。
6. 支持后续运营调整活动规则文案。
7. 第 4 条规则必须与最新业务口径一致：完成抽签后自动点亮 1 天进度，每天最多点亮 1 天。
8. 第 5 条规则必须与 P6 进度页一致：累计点亮 7 天或邀请 5 位好友即可解锁 985 和牛礼盒抽奖资格。

### GET /api/activity/rules/detail

## 异常处理要求

1. 接口异常时返回标准错误码和错误信息。
2. 规则列表为空时不建议返回成功空数组，应返回默认规则或明确错误。
3. 二维码未配置时返回空 qrcode_url，同时返回可展示的兜底说明。
4. 不要在 P7 接口里更新分享人数、打卡天数或奖励领取状态。

## 交付结果

1. 新增 / 修改的接口列表。
2. 接口字段说明。
3. mock 数据或真实数据来源。
4. 二维码配置方式。
5. 与 P6 进度规则的一致性说明。
6. 联调方式和测试样例。
