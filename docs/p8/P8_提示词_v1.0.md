# P8 H5 前端开发提示词

你是本项目的 **H5 前端开发 Agent**。

## 当前任务

只实现 **P8 大奖资格确认页 / 985 和牛礼盒抽奖资格详情页 / 开奖状态页**。

## 重要限制

1. 只处理 P8，不要开发 P1、P2、P4、P5、P6、P7 或其他页面的完整业务。
2. P8 只展示大奖资格、抽奖编号、开奖状态和企微二维码，不在前端执行开奖逻辑。
3. 抽奖编号必须来自接口，前端不能本地生成、不能写死。
4. 达标判断必须以后端返回为准，前端不要自己重新计算。
5. 不要新增未确认的业务规则。
6. 不要改动与 P8 无关的页面逻辑。
7. 不要私自改文案。
8. 顶部「连续打卡7天 大奖资格已解锁」需要支持后端配置；如果用户是通过邀请 5 人达标，不能错误固定成连续打卡 7 天。

## P8 已确认规则

1. 本页为大奖资格确认页。
2. 建议入口为 P6 里 985 和牛礼盒达标后的「去使用」，正式路由为 `/activity/grand-prize`。
3. 达标规则与 P6 保持一致：分享 5 个好友，或累计点亮 / 打卡 7 天，满足任一条件即可解锁大奖抽奖资格。
4. 页面展示 985 和牛礼盒抽奖资格。
5. 页面展示用户唯一抽奖编号，如「KY202406-0038」。
6. 抽奖编号由后端返回，前端只负责展示。
7. 页面展示三个权益说明：50元券企微领取、礼盒资格已解锁、开奖提醒企微通知。
8. 页面展示开奖状态，当前设计稿为「待开奖」态。
9. 开奖状态后续需支持待开奖、开奖中、已中奖、未中奖等状态。
10. 底部展示企微二维码，引导用户领取 50 元券、接收礼盒开奖通知、查看中奖结果。
11. 左上角返回按钮返回上一页；无历史栈时回 P6 我的奖励页。
12. 当前页不默认出现抽签按钮、奖励列表、商品推荐、再抽一次、签文结果或 AI 解签正文。
