# P8 QA Agent 任务卡

## 任务名称

验证 P8 大奖资格确认页是否符合设计稿、业务规则、接口状态和埋点要求。

## 测试范围

只测试 P8 相关页面逻辑，不测试 P1、P2、P4、P5、P6、P7 的完整业务闭环。

## 功能测试

1. 从 P6 达标后的 985 礼盒「去使用」能进入 P8；
2. P8 顶部主标题、资格确认卡、开奖状态卡、企微二维码模块按设计稿展示；
3. 左上角返回按钮可返回上一页；
4. 无历史栈直接打开 P8 时，返回按钮可回 P6 我的奖励页；
5. 页面展示 985 和牛礼盒抽奖资格；
6. 抽奖编号展示接口返回值，不由前端生成；
7. 三个权益说明完整展示：50元券企微领取、礼盒资格已解锁、开奖提醒企微通知；
8. 底部企微二维码正常展示。

## 业务规则测试

1. 分享 5 人达标后，P6 显示 985 已达标，点击可进入 P8；
2. 累计点亮 / 打卡 7 天达标后，P6 显示 985 已达标，点击可进入 P8；
3. 未达标用户访问 P8 时，不展示虚假抽奖编号；
4. 未达标用户访问 P8 时，可提示返回 P6 查看进度；
5. 通过打卡 7 天达标时，页面可展示「连续打卡7天 大奖资格已解锁」；
6. 通过邀请 5 人达标时，页面不应错误固定为「连续打卡7天」。

## 开奖状态测试

1. lottery_status=pending 时，展示「待开奖」和「开奖后将在此更新」；
2. lottery_status=drawing 时，展示「开奖中」或对应提示；
3. lottery_status=won 时，展示中奖结果或中奖说明；
4. lottery_status=not_won 时，展示未中奖说明；
5. lottery_status 缺失时有兜底文案，不出现空白状态。

## 异常测试

1. 资格详情接口失败时展示加载失败提示；
2. 抽奖编号为空时展示兜底提示，不本地生成编号；
3. 二维码地址为空时展示二维码未配置或兜底说明；
4. 二维码图片加载失败时展示失败占位；
5. 小屏手机页面可滚动，二维码和抽奖编号不被裁切；
6. 页面不出现抽签按钮、奖励列表、商品推荐、再抽一次按钮。

## 埋点测试

1. 进入 P8 触发 grand_prize_page_view；
2. 渲染成功触发 grand_prize_render_success；
3. 渲染失败触发 grand_prize_render_fail；
4. 未达标误入触发 grand_prize_not_qualified_view；
5. 开奖状态卡曝光触发 grand_prize_status_exposure；
6. 二维码曝光触发 grand_prize_qrcode_exposure；
7. 二维码加载失败触发 grand_prize_qrcode_load_fail；
8. 点击返回触发 grand_prize_back_click。

## 验收结论

测试通过后，确认 P8 可以作为大奖资格确认和开奖状态承接页上线；如抽奖编号、开奖状态、企微二维码仍为 mock，需要在验收结论里明确标注。
