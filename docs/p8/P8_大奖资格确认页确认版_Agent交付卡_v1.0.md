## 二、页面目标

P8 的目标是把「大奖资格已经解锁」这件事清楚地告知用户，并把后续开奖通知与福利领取沉淀到企微。

核心目标是：

1. 明确告诉用户已获得 985 和牛礼盒抽奖资格；
2. 明确展示用户自己的抽奖编号，增强可信度和仪式感；
3. 告知当前开奖状态和开奖时间；
4. 引导用户扫码添加企微，用于领取 50 元券、接收开奖提醒、查看中奖结果；
5. 与 P6 的达标逻辑保持一致，不能出现 P6 显示达标但 P8 显示未达标的情况；
6. 为后续开奖结果更新预留动态状态，不把「待开奖」写死为唯一状态。

## 六、接口需求

P8 必须依赖后端返回资格、抽奖编号和开奖状态，不能由前端自行判断或生成。

| 接口 | 触发时机 | 作用 |
|---|---|---|
| GET /api/grand-prize/qualification/detail | 进入 P8 时 | 获取大奖资格、达标方式、抽奖编号、开奖状态、开奖时间、企微二维码、页面文案 |
| GET /api/grand-prize/result | 开奖后进入页面或主动刷新时，可选 | 获取中奖结果、公示状态、中奖说明 |
| POST /api/tracking/event | 页面曝光、状态曝光、二维码曝光、异常 | 记录埋点事件 |

## 八、异常处理

| 异常情况 | 页面处理 |
|---|---|
| 资格详情接口失败 | 展示「资格信息加载失败，请稍后再试」，提供重试或返回 P6 |
| 用户未达标却进入 P8 | 展示「大奖资格暂未解锁」并引导返回 P6 查看进度；不展示抽奖编号 |
| 抽奖编号为空 | 展示「抽奖编号生成中，请稍后刷新」，记录异常，不前端伪造编号 |
| 开奖状态为空 | 按「待开奖」兜底展示，并记录 grand_prize_render_fail 或字段缺失埋点 |
| 开奖时间为空 | 展示兜底文案「开奖时间以活动页面公示为准」 |
| 二维码地址为空 | 展示「企微二维码暂未配置」，保留权益说明 |
| 二维码图片加载失败 | 展示加载失败占位，记录 grand_prize_qrcode_load_fail |
| 返回无历史栈 | 跳转 P6 我的奖励页 |
| 页面小屏溢出 | 支持纵向滚动，不压缩抽奖编号和二维码到不可读 |
| 文案与达标方式不一致 | 优先使用后端返回的 hero.title / qualify_desc，避免固定写死「连续打卡7天」 |

## 十、Agent 交付卡

### 10.1 H5 前端 Agent

任务：完成 P8 大奖资格确认页的视觉还原、资格动态展示、开奖状态展示、企微二维码承接和埋点触发。

必须实现：

1. 按 P8 主设计稿还原深红背景、顶部大标题、资格确认卡、开奖状态卡和企微二维码模块；
2. 左上角返回按钮可点击，返回上一页，无历史栈时回 P6 我的奖励页；
3. 页面展示大奖资格状态、达标文案、奖品名称和抽奖编号；
4. 抽奖编号必须来自接口，前端不可本地生成；
5. 顶部主文案支持后端配置，避免邀请 5 人达标时仍写成「连续打卡7天」；
6. 展示三个权益说明：50元券企微领取、礼盒资格已解锁、开奖提醒企微通知；
7. 展示开奖状态卡，支持待开奖、开奖中、已中奖、未中奖等状态；
8. 展示企微二维码模块，二维码资源支持配置或静态替换；
9. 未达标用户误入时，不展示抽奖编号，提示返回 P6 查看进度；
10. 页面支持纵向滚动和不同机型安全区；
11. 按埋点表触发页面曝光、渲染成功 / 失败、开奖状态曝光、二维码曝光、返回点击和异常事件；
12. 当前页不出现抽签按钮、奖励列表、商品推荐、再抽一次按钮。

### 10.2 后端 Agent

任务：提供 P8 所需的大奖资格、抽奖编号、开奖状态、企微二维码和动态文案能力。

必须实现 / 确认：

1. 根据用户 activity_id / session_id / openid 查询是否达标；
2. 达标规则与 P6 保持一致：分享 5 个好友或累计点亮 7 天满足任一即可；
3. 用户达标后返回唯一抽奖编号 lottery_no；
4. 抽奖编号由后端生成并持久化，不能每次进入页面变化；
5. 返回 qualify_type、qualify_desc、hero.title、hero.subtitle，支持不同达标方式文案；
6. 返回开奖状态 lottery_status，包括 status、status_text、draw_time_desc、publicity_desc；
7. 返回企微二维码 qrcode_url、qrcode_id 和扫码权益说明；
8. 未达标用户访问 P8 时返回 qualified=false 和回退 action；
9. 支持开奖后更新状态为 won / not_won，并返回对应结果说明；
10. 返回埋点所需 activity_id、session_id、lottery_no、qualify_type、lottery_status 等参数。

### 10.3 埋点 Agent
