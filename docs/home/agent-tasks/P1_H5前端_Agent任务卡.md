# P1 H5 前端 Agent 任务卡

# 项目背景

项目：高考考运摇签 + 牛肉福利领取 H5 活动  
页面：P1 活动首页 / 活动入口页  
状态：P1 已确认，可进入 Agent 开发拆分

## P1 已确认业务口径

1. 主按钮固定为「立即摇签」。
2. 新用户默认展示「我的摇签机会 1次」。
3. 签筒上的 5 支签不参与真实签文池，仅作为首页视觉装饰和后续动态效果展示。
4. 点击签筒与点击「立即摇签」逻辑一致，都触发摇签流程。
5. 「活动规则」跳转活动规则独立页面，不使用弹窗。
6. 「我的奖励」点击后跳转 P6 我的考运进度 / 我的奖励页面。
7. 分享行为不限制；每日最多奖励 3 次摇签机会；超过 3 次后继续允许分享，但不再增加机会。
8. 首页不设置「今日已摇签」按钮。
9. 有额外摇签机会时，主按钮不变，仍为「立即摇签」；只更新底部摇签机会数量。
10. P1 不处理活动结束态；活动结束后活动入口直接下架。


---

## 你的角色

你是本项目的 H5 前端 Agent，只负责 P1 活动首页的页面还原、状态渲染、按钮动作和跳转入口。

## 本次任务目标

完成 P1 活动首页的前端实现，让页面可以作为 H5 活动主入口，并能正确进入后续 P2 / P6 / P7 活动规则等页面。

> 入口口径：正式/测试入口页为 `/activity/home`。当前代码内部暂以 `home` 表示 P1，`?page=p1` 只作为早期截图预览兼容别名，不再作为测试环境入口。

## 实现范围

### 必须实现

1. 按 P1 设计稿还原首页视觉结构，包括：
   - 顶部「一举高中」
   - 「赢 985 和牛大礼包」
   - 中间签筒与 5 支签
   - 右侧「活动规则」
   - 主按钮「立即摇签」
   - 底部「我的奖励」「我的摇签机会 N次」「分享获取次数」

2. 主按钮固定展示：
   - 「立即摇签」

3. 新用户默认状态：
   - 底部展示「我的摇签机会 1次」

4. 签筒点击：
   - 点击签筒与点击「立即摇签」逻辑一致。

5. 点击「立即摇签」或签筒：
   - 调用正式抽签接口 `POST /api/draw/execute`。
   - 若接口返回成功，使用返回的 `draw_id` / 最新机会进入 P2。
   - 若接口返回 `no_chance` 或 `draw_chance <= 0`，提示：「今日机会已用完，分享可再得机会」。

6. 点击「活动规则」：
   - 跳转活动规则独立页面。
   - 不做弹窗。

7. 点击「我的奖励」：
   - 跳转 P6 我的考运进度 / 我的奖励页面。

8. 点击「分享获取次数」：
   - 打开分享引导 / 分享海报入口。
   - 分享行为不限制。
   - 每日最多奖励 3 次摇签机会。
   - 超过 3 次后继续允许分享，但不再增加机会，不做负向拦截提示。
   - 分享完成后调用 `POST /api/share/record` 记录分享行为并返回分享 token。
   - 好友通过分享进入并完成抽签后，原用户增加 1 个助力人数。

9. 底部「我的摇签机会 N次」：
   - 根据接口返回动态更新。
   - 有额外机会时，只更新数量，不改变主按钮文案。

10. 页面埋点：
    - 按埋点 Agent 定义触发前端埋点调用。

## 不要实现

1. 不要做「今日已摇签」按钮。
2. 不要把签筒上的 5 支签当作真实签文结果。
3. 不要在 P1 处理活动结束态。
4. 不要在 P1 提供「查看开奖结果」入口。
5. 不要把「活动规则」做成弹窗。
6. 不要在分享达到 3 次后阻止用户继续分享。
7. 不要私自新增未确认的页面和业务规则。

```ts
type P1ActivityState = {

  drawChance: number;
  shareRewardCountToday: number;

  rewardStatus?: string;
};
```

## 按钮动作映射

| 按钮 / 区域 | 前端动作 |
|---|---|
| 立即摇签 | 调 `POST /api/draw/execute`，成功后带 draw_id 进入 P2 |
| 签筒 | 同「立即摇签」 |
| 活动规则 | 跳转活动规则独立页面 |
| 我的奖励 | 跳转 P6 |
| 分享获取次数 | 打开分享引导 / 分享海报入口；分享完成后调 `POST /api/share/record` |

## 异常处理

| 场景 | 前端处理 |
|---|---|
| 首页状态接口失败 | 提示「网络开小差了，请稍后再试」，提供重试 |
| session 创建失败 | 页面可先展示，用户点击摇签时再次尝试创建 |
| 摇签机会为 0 | 点击后提示「今日机会已用完，分享可再得机会」 |
| 分享奖励次数已达 3 次 | 继续允许分享，不做负向拦截；不再增加机会 |
| P2 / P6 / P7 页面承接 | 文档中的“跳转”代表业务目标；当前代码仍可能通过 `currentPage` 本地状态切换 |
| 当前本地状态切页 | 文档中的“跳转”代表业务目标；当前代码仍可能通过 `currentPage` 本地状态切换 |

## 验收标准

1. 新用户进入 P1，底部展示「我的摇签机会 1次」。
2. 主按钮始终展示「立即摇签」。
3. 点击「立即摇签」且有机会时，进入 P2。
4. 点击签筒且有机会时，进入 P2。
5. 无机会时点击主按钮，出现「今日机会已用完，分享可再得机会」。
6. 点击「活动规则」进入独立规则页。
7. 点击「我的奖励」进入 P6。
8. 点击「分享获取次数」打开分享引导 / 分享海报入口。
9. 分享达到 3 次后，继续允许分享，但不再增加机会。
10. 页面不出现「今日已摇签」按钮。
11. P1 不出现「查看开奖结果」入口。
