# P1 QA 验收 Agent 任务卡

# 项目背景

项目：高考考运摇签 + 牛肉福利领取 H5 活动  
页面：P1 活动首页 / 活动入口页  
状态：P1 已确认，可进入 Agent 开发拆分

## P1 已确认业务口径

1. 主按钮固定为「立即摇签」。
2. 新用户默认展示「我的摇签机会 1次」。
3. 签筒上的 5 支签不参与真实签文池，仅作为首页视觉装饰和后续动态效果展示。
4. 点击签筒与点击「立即摇签」逻辑一致，都触发摇签流程。
5. 「活动规则」跳转活动规则独立页面，不使用弹窗。
6. 「我的奖励」点击后跳转 P6 我的考运进度 / 我的奖励页面。
7. 分享行为不限制；每日最多奖励 3 次摇签机会；超过 3 次后继续允许分享，但不再增加机会。
8. 首页不设置「今日已摇签」按钮。
9. 有额外摇签机会时，主按钮不变，仍为「立即摇签」；只更新底部摇签机会数量。
10. P1 不处理活动结束态；活动结束后活动入口直接下架。


---

## 你的角色

你是本项目的 QA Agent，只负责验证 P1 活动首页是否符合已确认业务口径、前端交互、接口状态和埋点要求。

## 本次任务目标

对 P1 进行功能验收、状态验收、异常验收和埋点验收，确保 P1 可交付给后续 P2 流程串联。

## 测试范围

页面：P1 活动首页 / 活动入口页

覆盖内容：

1. 页面展示
2. 摇签机会状态
3. 点击主按钮
4. 点击签筒
5. 活动规则跳转
6. 我的奖励跳转
7. 分享获取次数
8. 无机会提示
9. 活动结束不处理
10. 埋点事件

## 测试用例

### 1. 新用户首页展示

前置条件：

- 用户首次进入 H5。
- 后端返回 `drawChance = 1`。

预期结果：

- 页面展示 P1 设计稿内容。
- 主按钮为「立即摇签」。
- 底部展示「我的摇签机会 1次」。
- 不出现「今日已摇签」按钮。
- 不出现「查看开奖结果」入口。

---

### 2. 点击「立即摇签」

前置条件：

- `drawChance > 0`。

操作：

- 点击「立即摇签」。

预期结果：

- 前端调用机会校验。
- 校验通过后进入 P2 摇签动效页。
- 触发 `draw_entry_click` 埋点。

---

### 3. 点击签筒

前置条件：

- `drawChance > 0`。

操作：

- 点击中间签筒区域。

预期结果：

- 与点击「立即摇签」逻辑一致。
- 校验通过后进入 P2。
- 触发 `draw_tube_click` 埋点。
- 签筒上的 5 支签不作为真实签文结果。

---

### 4. 无摇签机会

前置条件：

- 后端返回 `drawChance = 0`。

操作：

- 点击「立即摇签」。

预期结果：

- 不进入 P2。
- 提示「今日机会已用完，分享可再得机会」。
- 触发 `draw_no_chance_view` 埋点。

---

### 5. 分享获取次数：第 1 次

前置条件：

- 今日分享奖励次数为 0。
- 当前摇签机会为 0 或 1。

操作：

- 点击「分享获取次数」并完成一次分享动作。

预期结果：

- 打开分享引导 / 分享海报入口。
- 分享后摇签机会 +1。
- 今日分享奖励次数变为 1。
- 触发 `share_get_chance_click`。
- 触发 `share_chance_add_success`。

---

### 6. 分享获取次数：累计 3 次

前置条件：

- 今日分享奖励次数小于 3。

操作：

- 连续完成 3 次有效分享奖励。

预期结果：

- 每次奖励 1 次摇签机会。
- 最多增加 3 次。
- 底部「我的摇签机会 N次」正确更新。
- 主按钮仍为「立即摇签」。

---

### 7. 分享超过 3 次

前置条件：

- 今日分享奖励次数已达 3。

操作：

- 继续分享。

预期结果：

- 继续允许分享。
- 不做负向拦截提示。
- 不再增加摇签机会。
- 不出现「今日分享获取次数已达上限」这类阻止分享的提示。
- 可选：后台记录内部事件，但不影响用户体验。

---

### 8. 活动规则跳转

操作：

- 点击「活动规则」。

预期结果：

- 跳转活动规则独立页面。
- 不使用弹窗。
- 触发 `rule_click` 埋点。

---

### 9. 我的奖励跳转

操作：

- 点击「我的奖励」。

预期结果：

- 跳转 P6 我的考运进度 / 我的奖励页面。
- 触发 `my_reward_click` 埋点。

---

### 10. 首页接口失败

前置条件：

- 模拟首页状态接口失败。

预期结果：

- 页面提示「网络开小差了，请稍后再试」。
- 提供重试入口。
- 触发 `activity_home_load_error` 埋点。

---

### 11. 活动结束态

前置条件：

- 活动已结束 / 活动入口已下架。

预期结果：

- P1 不处理活动结束态。
- 不在 P1 展示「查看开奖结果」入口。
- 如用户访问旧链接，应由上层入口下架或展示极简「活动已结束」提示。

## 不通过标准

出现以下任一情况，P1 不可通过验收：

1. 主按钮不是「立即摇签」。
2. 新用户默认机会不是 1 次。
3. 出现「今日已摇签」按钮。
4. 分享超过 3 次后阻止用户继续分享。
5. P1 中出现「查看开奖结果」入口。
6. 活动规则是弹窗而不是独立页面。
7. 我的奖励没有跳转 P6。
8. 签筒 5 支签被当作真实签文结果。
9. 无机会时仍进入 P2。
10. 关键点击没有埋点。

## 交付物

QA Agent 需要输出：

1. P1 验收结果。
2. 通过 / 不通过结论。
3. 不通过问题列表。
4. 复现步骤。
5. 修复建议。
6. 回归验证结果。
