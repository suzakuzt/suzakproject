# P1 活动首页确认版 + Agent 交付卡 v1.0

> 页面状态：已确认，可交付 Agent  
> 页面名称：P1 活动首页 / 活动入口页  
> 项目：高考考运摇签 + 牛肉福利领取 H5 活动

---

## 一、P1 已确认业务口径

| 序号 | 确认项 | 最终结论 |
|---|---|---|
| 1 | 主按钮文案 | 固定为「立即摇签」 |
| 2 | 新用户默认机会 | 新用户默认展示「我的摇签机会 1次」 |
| 3 | 签筒 5 支签 | 不参与真实签文，仅作为首页视觉装饰和后续动态效果展示 |
| 4 | 签筒点击 | 点击签筒也触发摇签，与点击「立即摇签」一致 |
| 5 | 活动规则 | 跳转活动规则独立页面 |
| 6 | 我的奖励 | 点击后跳转 P6 |
| 7 | 分享获取次数 | 每分享 1 次获得 1 次摇签机会，每日最多 3 次 |
| 8 | 今日已摇签按钮 | 首页不设置「今日已摇签」按钮 |
| 9 | 有额外机会时按钮 | 主按钮不变，仍为「立即摇签」；只更新底部摇签机会数量 |

---

## 二、页面目标

P1 只负责活动入口，不承载复杂业务。核心目标是：

1. 告诉用户活动利益点：赢 985 和牛大礼包；
2. 建立「一举高中 / 金榜题名」的高考好运氛围；
3. 引导用户点击「立即摇签」进入 P2；
4. 提供「活动规则」「我的奖励」「分享获取次数」三个辅助入口。

---

## 三、页面元素与业务含义

| 页面元素 | 业务含义 | 备注 |
|---|---|---|
| 一举高中 | 活动主题 | 固定展示 |
| 赢 985 和牛大礼包 | 核心利益点 | 固定展示 |
| 签筒 + 5 支签 | 首页主视觉 / 动态效果素材 | 不进入真实签文池 |
| 立即摇签 | 主 CTA | 校验机会后进入 P2 |
| 活动规则 | 跳转规则页 | 独立页面 |
| 我的奖励 | 跳转 P6 | 展示奖励 / 进度 / 资格 |
| 我的摇签机会 N次 | 动态展示剩余机会 | 新用户默认 1 次 |
| 分享获取次数 | 获取额外摇签机会入口 | 每日最多 +3 次 |分享行为不限制；每日最多奖励 3 次摇签机会，超过 3 次后继续允许分享，但不再增加机会 |

## 四、按钮动作

| 按钮 / 区域 | 点击动作 | 规则 |
|---|---|---|
| 立即摇签 | 校验剩余机会，成功后进入 P2 摇签动效页 | 机会 > 0 才进入 P2 |
| 签筒 | 同「立即摇签」 | 增强互动 |
| 活动规则 | 跳转活动规则独立页面 | 不使用弹窗 |
| 我的奖励 | 跳转 P6 | P6 按当前设计编号执行 |
| 分享获取次数 | 打开分享引导 / 分享海报 | 每分享 1 次，机会 +1；每日最多 +3 |分享行为不限制；每日最多奖励 3 次摇签机会，超过 3 次后继续允许分享，但不再增加机会 |

---

## 五、状态逻辑

| 用户状态 | 首页主按钮 | 底部机会展示 | 点击主按钮后 |
|---|---|---|---|
| 新用户，未摇签 | 立即摇签 | 我的摇签机会 1次 | 进入 P2 |
| 已摇签，机会为 0 | 立即摇签 | 我的摇签机会 0次 | 提示分享可获得机会 |
| 已分享 1 次 | 立即摇签 | 我的摇签机会 1次 | 进入 P2 |
| 已分享 3 次 | 立即摇签 | 我的摇签机会 3次 | 进入 P2，之后按剩余次数递减 |

> 注意：首页不展示「今日已摇签」按钮；除活动结束等特殊状态外，主按钮保持「立即摇签」。

## 六、接口需求

| 接口 | 触发时机 | 作用 |
|---|---|---|
| POST /api/h5/session/create | 首页首次进入 | 创建或恢复 activity_session_id |
| GET /api/activity/state | 首页加载 | 获取剩余摇签机会、分享次数、活动状态、奖励状态 |
| POST /api/draw/chance/check | 点击「立即摇签」或签筒 | 校验当前是否有摇签机会 |
| POST /api/share/chance/add | 分享完成后 | 增加 1 次摇签机会，每日最多 3 次 | 分享行为不限制；每日最多奖励 3 次摇签机会，超过 3 次后继续允许分享，但不再增加机会 |
| POST /api/tracking/event | 页面曝光和点击 | 记录埋点事件 |

## 七、埋点事件

| 事件名 | 触发时机 | 关键参数 |
|---|---|---|
| h5_activity_visit | 进入 H5 首页 | activity_id、session_id、channel、invite_code |
| activity_home_view | 首页渲染完成 | session_id、draw_chance、share_count_today |
| draw_entry_click | 点击「立即摇签」 | session_id、draw_chance |
| draw_tube_click | 点击签筒 | session_id、draw_chance |
| rule_click | 点击活动规则 | session_id |
| my_reward_click | 点击我的奖励 | session_id |
| share_get_chance_click | 点击分享获取次数 | session_id、share_count_today |
| share_chance_add_success | 分享后机会增加成功 | session_id、share_count_today、draw_chance |
| draw_no_chance_view | 无摇签机会提示曝光 | session_id |
| activity_home_load_error | 首页加载失败 | error_code、error_msg |

## 八、异常处理

| 异常情况 | 页面处理 |
|---|---|
| 首页接口失败 | 提示「网络开小差了，请稍后再试」，提供重试 |
| session 创建失败 | 页面可先展示，点击摇签时重试创建 |
| 摇签机会为 0 | 点击「立即摇签」后提示「今日机会已用完，分享可再得机会」 |
| 分享次数已达 3 次 | 每日最多奖励 3 次摇签机会；超过 3 次后继续允许分享，只是不再增加摇签机会；不做负向拦截提示。」 |
| 活动已结束 | P1 不处理活动结束态。活动结束后，这个活动入口直接下架。 |

## 九、Agent 交付卡

### 9.1 H5 前端 Agent

任务：完成 P1 首页展示、状态渲染、按钮动作和跳转。

必须实现：

1. 首页视觉按设计稿还原；
2. 主按钮固定展示「立即摇签」；
3. 新用户默认展示「我的摇签机会 1次」；
4. 点击签筒与点击主按钮逻辑一致；
5. 活动规则跳转独立页面；
6. 我的奖励跳转 P6我的考运进度页面；
7. 分享获取次数打开分享引导 / 分享海报；
8. 底部摇签机会根据接口返回动态更新。

### 9.2 后端 Agent

任务：提供 P1 所需活动状态、机会校验、分享加次数能力。

必须实现：

1. 创建 / 恢复 activity_session_id；
2. 返回用户当前剩余摇签机会；
3. 返回今日分享获取次数；
4. 分享 1 次增加 1 次摇签机会；
5. 每日最多通过分享增加 3 次；
6. 摇签前校验机会是否足够；
7. 机会使用后扣减。

### 9.3 埋点 Agent

任务：按埋点表实现 P1 页面曝光、点击、分享、异常事件。

必须覆盖：

1. 首页访问；
2. 首页渲染成功；
3. 点击立即摇签；
4. 点击签筒；
5. 点击活动规则；
6. 点击我的奖励；
7. 点击分享获取次数；
8. 分享增加机会成功；
9. 无机会提示；
10. 首页加载失败。

### 9.4 QA Agent

任务：验证 P1 业务逻辑是否符合确认口径。

必须测试：

1. 新用户进入展示 1 次摇签机会；
2. 点击「立即摇签」进入 P2；
3. 点击签筒进入 P2；
4. 无机会时点击主按钮出现提示；
5. 分享 1 次后机会 +1；
6. 分享 3 次后机会 +3；
7. 超过 3 次后不再增加；
8. 活动规则跳转独立页面；
9. 我的奖励跳转 P6；
10. 首页不出现「今日已摇签」按钮。
