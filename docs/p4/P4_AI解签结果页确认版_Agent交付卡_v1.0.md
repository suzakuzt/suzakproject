## 二、页面目标

P4 负责承接 P2 的「问小璞」动作，把抽签结果进一步解释成更有仪式感的 AI 解签内容，同时把用户自然引导到牛肉福利领取动作。

核心目标是：

1. 让用户看到完整的 AI 解签结果；
2. 强化「高考好运」与「牛气补给」之间的关联；
3. 通过牛肉商品露出建立福利感和品牌关联；
4. 引导用户点击「领取专属福利」；
5. 为后续优惠券、企微、商品详情、小程序跳转等能力预留接口，不在 P4 当前页面直接展开复杂业务。

## 五、状态逻辑

| 用户状态 / 页面状态 | 页面展示 | 点击动作 |
|---|---|---|
| AI 解签加载中 | 展示「AI 解签中」和短句思考过程 / 点状动效 | 暂不允许点击领取 |
| AI 解签加载成功 | 展示 AI 解签文案、主题标签、牛肉推荐卡、领取按钮 | 可点击领取 |
| AI 解签加载失败 | 提示「解签结果加载失败，请稍后再试」 | 可重试或返回上一页 |
| 商品推荐加载失败 | 保留 AI 解签结果，商品卡展示默认兜底内容 | 领取按钮仍按后端返回结果判断 |
| P5 弹窗打开中 | P4 保持背景页，弹出手机号领券层 | 防止重复打开 |
| P5 提交成功 | 展示成功反馈并按 action 承接小程序 | 成功领取后的优惠券才可在 P6 我的奖励展示 |
| P5 提交失败 | 在 P5 内提示失败原因 | 可重试 |
| 已领取 | 按后端状态展示「已领取」或进入已领取结果 | 不重复发放福利 |

## 六、接口需求

| 接口 | 触发时机 | 作用 |
|---|---|---|
| GET /api/explain/detail | 进入 P4 时 | 获取 AI 解签结果、主题标签、牛肉推荐卡、领取状态 |
| POST /api/benefit/claim | P5 输入手机号后点击「去领取」 | 领取当前解签结果对应的专属福利，保存手机号和领取 token |
| POST /api/tracking/event | 页面曝光、点击、异常 | 记录埋点事件 |

### 6.1 AI 解签结果接口建议字段

当前实现由后端代理 DeepSeek `/chat/completions` 生成 AI 解签文案，前端不接触 API Key。接口应返回 `explainLines` / `explain_lines`、`thinkingProcess`、`themeText` 和 `ai.promptVersion`；未配置 Key、超时或返回异常时，后端使用本地兜底文案，页面保持可展示。

```json
{
  "activity_id": "gaokao_lucky_sign_2026",
  "session_id": "string",
  "draw_id": "string",
  "explain_id": "string",
  "sign_type": "金榜题名签",
  "sign_level": "上上签",
  "title": "AI解签结果",
  "explain_lines": [
    "锦绣前程，步步生花",
    "实力如锦，终成佳绩",
    "心之所向，金榜题名"
  ],
  "theme_text": "高考好运 × 牛气补给",
  "product": {
    "product_id": "string",
    "product_name": "和牛 · 锦绣前程板腱",
    "product_image": "https://example.com/product.png"
  },
  "benefit": {
    "benefit_id": "string",
    "claim_status": "unclaimed",
    "claim_button_text": "领取专属福利"
  }
}
```

### 6.2 P5 福利领取接口建议字段

```json
{
  "session_token": "string",
  "draw_id": "string",
  "reward_code": "coupon_10",
  "mobile": "13812345678",
  "claim_channel": "h5"

```

P4 按钮只负责打开 P5，不直接发送以上请求。

## 八、异常处理

| 异常情况 | 页面处理 |
|---|---|
| AI 解签接口失败 | 提示「解签结果加载失败，请稍后再试」，提供重试 |
| AI 解签文案为空 | 使用兜底提示「今日好运正在生成，请稍后再试」 |
| 商品图片加载失败 | 使用默认牛肉福利占位图 |
| 商品名称为空 | 展示「专属牛气补给」 |
| P5 领券接口失败 | P5 内提示「福利领取失败，请稍后再试」 |
| 用户重复点击领取 | loading 中禁用按钮，防重复提交 |
| 用户已领取 | 按后端状态展示已领取状态，避免重复发放 |
| 埋点失败 | 不阻断页面主流程，前端可静默失败或重试 |

## 九、视觉与切图建议

| 模块 | 建议处理方式 | 说明 |
|---|---|---|
| 红色丝绒背景 | 可作为静态背景图 | 不承载业务，不需要动态 |
| 米金吊牌主卡 | 可作为页面主视觉背景 / 容器背景 | 注意保留顶部挂绳和底部绳结 |
| 标题牌「AI解签结果」 | 建议前端文字渲染，或作为静态标题切图 | 如后续标题不变，可静态 |
| AI 解签正文 | 必须前端文本渲染 | 不能整块切成图片，便于动态替换 |
| 主题标签 | 可前端文本渲染 | 推荐保留图标和分割装饰 |
| 牛肉图片 | 独立图片位 | 支持后端返回图片地址 |
| 商品名称 | 前端文本渲染 | 支持动态替换 |
| 领取按钮 | 可用 CSS 还原或按钮切图 + 文本覆盖 | 文案固定为「领取专属福利」 |

## 十、Agent 交付卡

### 10.1 H5 前端 Agent

任务：完成 P4 AI 解签结果页展示、动态渲染、福利领取入口和基础状态处理。

必须实现：

1. 页面视觉按 P4 主设计稿还原；
2. 固定展示标题「AI解签结果」；
3. AI 解签文案支持动态渲染，不写死到图片里；
4. 固定展示主题标签「高考好运 × 牛气补给」；
5. 牛肉商品图、商品名称支持动态配置；
6. 主按钮固定展示「领取专属福利」；
7. 点击按钮后打开 P5 手机号领券弹窗，P5 提交成功后再生成领取记录；
8. 支持 AI 解签加载态、失败态、领取中、领取成功、领取失败、已领取状态；
9. 保留后续接接口位置，不要把业务写死；
10. 按埋点表触发页面曝光、商品曝光、点击、成功、失败事件。

### 10.2 后端 Agent

任务：提供 P4 所需 AI 解签结果、推荐商品和福利领取能力。

必须实现：

1. 根据 draw_id / explain_id 返回 AI 解签结果；
2. 返回多行解签文案 explain_lines；
3. 返回主题文案 theme_text；
4. 返回推荐商品 product_id、product_name、product_image；
5. 返回福利 benefit_id、claim_status、claim_button_text；
6. 支持 P5 手机号领券接口；
7. 处理已领取、防重复领取、异常状态；
8. 返回埋点所需 activity_id、session_id、draw_id、explain_id、product_id、benefit_id 等字段。

### 10.3 埋点 Agent

任务：按埋点表实现 P4 页面曝光、渲染、商品曝光、领取点击、领取成功 / 失败、异常事件。

必须覆盖：

1. P4 AI 解签结果页曝光；
2. P4 渲染成功；
3. AI 解签加载失败；
4. 牛肉推荐卡曝光；
5. 点击领取专属福利；
6. 福利领取成功；
7. 福利领取失败；
8. 用户返回上一页。

### 10.4 QA Agent

任务：验证 P4 页面逻辑是否符合当前设计口径。

必须测试：

1. 从 P2 点击「问小璞」后能进入 P4；
2. 页面标题固定展示「AI解签结果」；
3. AI 解签文案支持动态替换；
4. 主题标签展示「高考好运 × 牛气补给」；
5. 牛肉商品图正常展示；
6. 商品名称支持动态替换；
7. 点击「领取专属福利」能打开 P5 手机号领券弹窗；
8. 领取中防重复点击；
9. 领取成功后有明确承接；
10. 领取失败时有异常提示；
11. AI 解签加载失败时有异常提示；
12. 页面不出现未确认的再抽一次、分享海报、我的奖励列表、企微二维码、券列表等模块。

## 十一、交付状态

P4 最新口径：点击「领取专属福利」只打开 P5 手机号领券弹窗，P5 输入手机号并提交成功后才生成领取记录；P6 我的奖励页通过 `display_rewards` 展示奖励状态，已领取券显示「去领取」，未领取券显示「未领取」。P4 不直接展示 P6 奖励列表，也不在前端伪造领取结果。牛肉商品卡是否允许点击、AI 解签结果是否由真实 AI 生成，后续可继续按接口配置细化。
