# P4 QA Agent 任务卡

> 页面名称：P4 AI 解签结果页 / 福利承接页  
> 项目：高考考运摇签 + 牛肉福利领取 H5 活动  
> 任务角色：QA Agent

---

## 一、当前任务

验证 P4 页面是否符合当前主设计稿和业务口径，包括页面展示、动态数据、按钮动作、异常状态、埋点事件和未确认模块限制。

---

## 二、测试范围

只测试 P4：

1. AI 解签结果展示；
2. 牛肉推荐卡展示；
3. 领取专属福利按钮；
4. 加载态 / 失败态 / 已领取状态；
5. 埋点事件；
6. P2 到 P4 的入口链路。

不测试：

1. P1 首页完整流程；
2. P6 我的奖励完整流程；
3. 企微二维码领取完整流程；
4. 商品详情页完整流程；
5. 活动规则页。

---

## 三、核心用例

| 用例 | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| P4 正常进入 | P2 已生成 draw_id | 点击「问小璞」 | 进入 P4 |
| 标题展示 | P4 正常加载 | 查看页面 | 展示「AI解签结果」 |
| AI 文案展示 | 接口返回 explain_lines | 查看正文 | 多行文案正常展示 |
| AI 文案动态替换 | 修改 mock 文案 | 刷新页面 | 页面展示新文案 |
| 主题标签展示 | P4 正常加载 | 查看标签区 | 展示「高考好运 × 牛气补给」 |
| 商品图展示 | 接口返回 product_image | 查看商品卡 | 图片正常显示 |
| 商品名称动态替换 | 修改 product_name | 刷新页面 | 商品名称正确变化 |
| 领取按钮展示 | 未领取状态 | 查看按钮 | 展示「领取专属福利」 |
| 打开 P5 | 点击领取专属福利 | 点击按钮 | 打开 P5 手机号领券弹窗，不直接发领券请求 |
| 领取成功 | P5 输入正确手机号并提交，接口成功 | 点击去领取 | 成功领取后的优惠券才会出现在 P6 |
| 领取失败 | 点击领取，接口失败 | 点击按钮 | 提示「福利领取失败，请稍后再试」 |
| 防重复点击 | 领取接口 pending | 连续点击按钮 | 只发起一次领取请求 |
| 已领取状态 | claim_status=claimed | 进入页面 | 不重复发放福利 |
| 解签加载失败 | explain 接口失败 | 进入页面 | 提示「解签结果加载失败，请稍后再试」 |
| DeepSeek 正常 | 配置 `DEEPSEEK_API_KEY` 后进入 P4 | 查看接口响应 | 返回 `provider=deepseek`、`promptVersion`、三行合规签文和可展示 `thinkingProcess` |
| DeepSeek 兜底 | 未配置 Key、超时或接口异常 | 进入 P4 | 返回 fallback 文案，页面仍显示「AI 解签中」loading 后正常渲染 |
| 商品图失败 | 图片 URL 失效 | 进入页面 | 展示兜底图 |
| 未确认模块检查 | P4 正常加载 | 查看页面 | 不出现再抽一次、分享海报、我的奖励列表、企微二维码、券列表 |

## 四、埋点验收

| 事件名 | 验收方式 |
|---|---|
| ai_explain_page_view | 首次进入 P4 触发一次 |
| ai_explain_render_success | 数据渲染完成后触发一次 |
| ai_explain_load_fail | AI 解签接口失败时触发 |
| ai_explain_product_exposure | 商品卡曝光时触发一次 |
| exclusive_benefit_click | 点击领取按钮时触发 |
| exclusive_benefit_claim_success | 领取成功时触发 |
| exclusive_benefit_claim_fail | 领取失败时触发 |
| ai_explain_page_back | 返回上一页时触发 |

## 五、兼容性检查

建议至少覆盖：

1. iPhone 375 宽；
2. iPhone 390 宽；
3. iPhone 414 宽；
4. Android 360 宽；
5. Android 412 宽；
6. 微信内置浏览器；
7. 普通移动端浏览器。

重点检查：

1. 吊牌主卡是否被裁切；
2. 按钮是否超出安全区；
3. 商品图是否变形；
4. 多行 AI 文案是否溢出；
5. 页面高度不足时是否可滚动；
6. 长商品名是否换行或省略合理。

---

## 六、回归检查

1. P1 首页按钮和文案不受影响；
2. P2「问小璞」入口仍可点击；
3. P2「点击分享海报」入口不受影响；
4. P4 不影响已有奖励页；
5. 全局埋点方法不破坏其他页面。
