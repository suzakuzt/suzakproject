# P6 H5 前端开发提示词 v1.1

你是本项目的 **H5 前端开发 Agent**。

## 当前任务

只实现 **P6 我的奖励页 / 考运进度页 / 奖励中心页**。

本页基于 P6 主设计稿开发，核心负责展示用户分享进度、摇签打卡 / 点亮进度、已领取奖励、985 礼盒资格、精选好物推荐，并提供「再抽一次」入口。

---

## 重要限制

1. 只处理 P6，不要开发 P1、P2、P4、P5、活动规则页或其他页面完整业务。
2. 不要新增未确认的业务规则。
3. 不要私自新增「分享好友」按钮、企微二维码、领取成功弹窗、AI 解签正文、分享海报弹窗。
4. 不要把 `已分享 0/5 人`、`1-7 天进度`、奖励金额、折扣、商品文案整块写死到图片里。
5. 用户完成摇签后，后端会记录为打卡 / 点亮；每天最多点亮 1 天，P6 前端只读取展示，不自行累加。
6. 优惠券 / 折扣券按 `display_rewards` 展示；已领取显示「去领取」，未领取显示「未领取」。
7. 普通奖励固定展示 5 个配置位，不因重复领取同券码而新增卡片；同类奖励已领取时可使用最新一条领取记录的 `claim_no` / `claim_token` 承接小程序券包。
8. 985 礼盒资格卡始终展示，并固定排在最后一个。
9. 已领取券 + 985 礼盒卡总数 ≤ 3 时只显示一排；4-6 时显示两排。
10. 985 礼盒未达标展示「未达标」；分享 5 人或点亮 7 天任一达标后展示「去使用」。
11. 券类奖励点击「去领取」跳小程序券包页 / 我的券包页。
12. 精选好物点击「去看看」跳具体商品详情页。
13. 底部「再抽一次」跳回 P1 活动首页界面，正式/测试目标为 `/activity/home`。
14. 右上角「活动规则」跳 P7 活动规则独立页，正式路由为 `/activity/rules`。
15. 左上角返回按钮当前按返回上一页处理；如果无历史栈，则回 P1 首页。
16. 不要改动与 P6 无关的页面。
17. 不要私自改文案。

## P6 已确认规则

1. 本页为我的奖励页 / 考运进度页 / 奖励中心页。
2. 当前建议从 P1「我的奖励」入口进入 P6。
3. 分享完成后后端记录分享人数，P6 展示「已分享 X/5 人」。
4. 只要完成摇签，后端记录打卡 / 点亮；每天最多点亮 1 天，P6 展示 1-7 天点亮进度。
5. 分享 5 个好友，或累计点亮 7 天，赢取 985 和牛礼盒抽奖资格。
6. 优惠券 / 折扣券领取后才在「我的奖励」展示。
7. 我的奖励最多 6 张卡：已领取券类奖励 + 固定最后一张 985 礼盒资格卡。
8. 3 张及 3 张以内只显示 1 排；4-6 张显示 2 排。
9. 985 礼盒未达标展示「未达标」；达标后展示「去使用」。
10. 券类奖励「去领取」跳小程序券包页。
11. 精选好物「去看看」跳具体商品详情页。
12. 「再抽一次」跳 P1 活动首页，正式/测试目标为 `/activity/home`。
13. 「活动规则」跳 P7 活动规则页，正式路由为 `/activity/rules`。
14. 当前页不默认出现 AI 解签正文、领取成功弹窗、企微二维码、分享海报弹窗。

### 2. 考运进度动态渲染

从接口 / mock 中读取：

```ts
type Progress = {
  shared_count: number
  share_target: number
  lit_days: number
  light_target: number
  completed_days: number[]
  gift_qualified: boolean
  gift_status: 'not_qualified' | 'qualified' | string
  progress_desc: string
}
```

渲染要求：

1. `已分享 X/5 人` 中的 X 和 5 不要写死；
2. 第 1-7 天通过循环渲染；
3. `completed_days` 内的天数显示为已点亮；
4. 已点亮状态在原有圆点基础上标红，不额外叠加勾选按钮；
5. 未点亮状态显示米金未完成态；
6. 数据缺失时用 0 分享、0 点亮兜底，不白屏；
7. 不在 P6 里新增打卡按钮；打卡由摇签流程完成后后端记录。

### 3. 我的奖励动态渲染

从接口 / mock 中读取：

```ts
type ClaimedReward = {
  reward_id: string
  reward_type: 'coupon' | 'discount_coupon' | string
  title: string
  unit_text?: string
  desc: string
  status: 'unused' | 'used' | 'expired' | 'pending' | string

  }

  reward_type: 'gift_lottery_qualification' | string
  title: string
  desc: string
  status: 'not_qualified' | 'qualified' | 'unused' | string
  button_text: '未达标' | '去使用' | string
  image_url?: string
  action?: {
    type: 'gift_qualification_detail' | 'mini_program_coupon_package' | string
    target?: string
  }
}
```

展示要求：

1. 渲染 `display_rewards` 作为 P6 实际展示列表；
2. 未领取券展示「未领取」且不可使用；
3. 985 礼盒 `gift_reward` 始终展示，且固定追加到最后一个；
4. 当前展示卡片总数 ≤ 3 时只显示 1 排；4-6 时显示 2 排；
5. 金额券支持 10 / 20 / 30 等动态金额；
6. 折扣券支持 9 折、7.5 折等动态展示；
7. 券类奖励 `unused` 时展示「去领取」，点击跳小程序券包页；
8. 券类奖励 `used / expired / pending` 不按可用奖励处理；
9. 985 礼盒 `not_qualified` 时展示「未达标」，不可使用；
10. 985 礼盒 `qualified / unused` 时展示「去使用」。
