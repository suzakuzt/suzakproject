# P6 QA Agent 任务卡 v1.1

你是本项目的 **QA Agent**。

## 当前任务

验证 P6 我的奖励页 / 考运进度页是否符合最新业务规则。

---

## 核心规则验收

### 1. 页面基础

1. 从 P1「我的奖励」入口进入 P6，页面正常展示。
2. 页面视觉符合 P6 主设计稿。
3. 左上角返回按钮可返回上一页，无历史栈时回 P1。
4. 右上角「活动规则」跳活动规则页。
5. 页面不出现 AI 解签正文、领取成功弹窗、企微二维码、分享海报弹窗。

### 2. 分享进度

1. shared_count = 0 时展示「已分享 0/5 人」。
2. shared_count = 1 时展示「已分享 1/5 人」。
3. shared_count = 5 时展示「已分享 5/5 人」。
4. 分享达标后，985 礼盒资格应达标。
5. 分享记录由后端返回，前端刷新后能正确读取。

### 3. 摇签打卡 / 点亮进度

1. 用户完成摇签后，后端记录打卡 / 点亮；每天最多点亮 1 天。
2. lit_days = 0 时，7 个节点均为未点亮。
3. lit_days = 3 时，第 1-3 天点亮，第 4-7 天未点亮。
4. lit_days = 7 时，7 个节点全部点亮。
5. 点亮达标后，985 礼盒资格应达标。
6. P6 前端不能自行累加点亮天数，只能读取接口数据，并且重复抽签不应让同一天点亮多次。

### 4. 我的奖励

1. 未领取的优惠券 / 折扣券展示为「未领取」且不可使用。
2. 已领取的优惠券 / 折扣券才展示。
3. 985 礼盒资格卡始终展示。
4. 985 礼盒资格卡始终排在最后一个。
5. 已领取券 0 张时，只展示 985 礼盒卡或后端兜底状态。
6. 已领取券 2 张 + 985 时，总共 3 张，只显示 1 排。
7. 已领取券 3 张 + 985 时，总共 4 张，显示 2 排。
8. 已领取券 5 张 + 985 时，总共 6 张，显示 2 排。
9. 允许按 `display_rewards` 展示未领取的 10 / 20 / 30 / 9 折 / 7.5 折，但必须显示「未领取」，不能按已领取券处理。

### 5. 985 礼盒状态

1. 分享未达标且点亮未达标时，985 礼盒展示「未达标」。
2. 点击「未达标」不可进入使用流程，可提示达成条件。
3. 分享达标或点亮达标时，985 礼盒展示「去使用」。
4. 点击「去使用」按后端 action 进入资格承接。

### 6. 跳转逻辑

1. 券类奖励点击「去领取」跳小程序券包页 / 我的券包页。
2. 精选好物点击「去看看」跳具体商品详情页。
3. 底部「再抽一次」跳 P1 活动首页。
4. 活动规则跳活动规则页。
5. 券包跳转失败时有提示。
6. 商品详情跳转失败时有提示。
7. 活动规则跳转失败时有提示。

### 7. 埋点验证

1. 页面曝光事件触发。
2. 渲染成功 / 加载失败事件触发。
3. 进度模块曝光参数正确。
4. 奖励卡曝光只包含已展示卡。
5. 985 未达标点击和达标点击事件区分正确。
6. 券包、商品详情、再抽一次、活动规则、返回点击事件均触发。
