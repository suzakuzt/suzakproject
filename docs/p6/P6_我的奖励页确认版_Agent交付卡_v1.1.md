## 二、页面目标

P6 负责把用户在活动中的「进度、奖励、商品承接、再抽一次入口」集中展示出来。

核心目标是：

1. 让用户明确知道自己当前已分享人数、累计点亮天数，以及距离 985 礼盒资格还差多少；
2. 展示 P6 奖励配置中的普通奖励，已领取奖励可用，未领取奖励只展示「未领取」状态；
3. 固定露出 985 和牛礼盒抽奖资格卡，并根据达标状态展示「未达标」或「去使用」；
4. 用「精选好物推荐」承接具体商品详情页转化；
5. 用「再抽一次」把用户带回 P1 首页继续活动链路。

## 四、按钮动作

| 按钮 / 区域 | 点击动作 | 规则 |
|---|---|---|
| 返回按钮 | 返回上一页 | 当前按 history back；如果没有历史栈，回到 P1 活动首页 |
| 活动规则 | 进入活动规则页 | 跳独立规则页，不在 P6 弹复杂说明 |
| 券类奖励「去领取」 | 跳转小程序券包页 / 我的券包页 | 用小程序跳转能力承接，前端不要弹占位 |
| 985 礼盒「未达标」 | 不可点击或点击提示未达标 | 提示「分享 5 个好友或累计点亮 7 天后可获得抽奖资格」 |
| 985 礼盒「去使用」 | 跳转 985 礼盒资格承接 | 达标后可点击，承接页由后端 action 返回；可为券包页 / 资格详情页 |

| 奖励卡主体 | 默认只展示 | 当前只以按钮作为明确点击入口 |
| 装饰线 / 边框 / 光效 | 无动作 | 仅视觉装饰 |

## 五、状态逻辑

| 用户状态 / 页面状态 | 页面展示 | 点击动作 |
|---|---|---|
| 页面加载中 | 展示骨架屏或 loading | 暂不允许重复点击 |
| 页面加载成功 | 展示考运进度、已领取奖励、985 礼盒卡、商品推荐、再抽一次 | 正常点击 |
| 页面加载失败 | 提示「奖励信息加载失败，请稍后再试」 | 可重试或返回首页 |
| 分享进度为空 | 展示「已分享 0/5 人」兜底 | 不影响其他模块 |
| 分享未达标 | 985 礼盒按分享条件未达标处理 | 继续看点亮是否达标 |
| 分享达标 | 985 礼盒资格达标 | 985 卡展示「去使用」 |
| 点亮天数为空 | 展示 0 天点亮兜底 | 1-7 天均未点亮 |
| 用户已摇签 | 后端记录当天 / 本次打卡点亮 | P6 下次进入读取最新点亮进度 |
| 点亮未达标 | 985 礼盒按点亮条件未达标处理 | 继续看分享是否达标 |
| 点亮达标 | 985 礼盒资格达标 | 985 卡展示「去使用」 |
| 分享和点亮都未达标 | 985 礼盒卡展示「未达标」 | 不允许使用 |
| 分享或点亮任一达标 | 985 礼盒卡展示「去使用」 | 可进入资格承接 |
| 没有已领取券 | 我的奖励区仍展示配置内普通奖励 | 普通奖励按钮显示「未领取」，985 礼盒仍排最后 |
| 已领取券 ≤ 2 张 | 已领取券 + 985 礼盒卡总数 ≤ 3，展示 1 排 | 正常点击可用奖励 |
| 已领取券 3-5 张 | 已领取券 + 985 礼盒卡总数 4-6，展示 2 排 | 正常点击可用奖励 |
| 券类奖励可用 | 展示「去领取」 | 点击跳小程序券包页 |
| 券类奖励已使用 | 可展示已使用态或由后端过滤 | 不允许重复使用 |
| 券类奖励已失效 | 可展示已失效态或由后端过滤 | 不允许使用 |
| 券类奖励跳转失败 | 提示「暂时无法打开券包，请稍后再试」 | 可重试 |
| 商品推荐为空 | 隐藏推荐区或展示后端兜底商品 | 不留空白断层 |
| 商品跳转失败 | 提示「暂时无法打开商品，请稍后再试」 | 可重试 |
| 再抽一次 | 跳转 P1 首页 | 不在 P6 判断抽签结果 |
| 用户重复点击 | 当前按钮 loading / 防抖 | 禁止重复请求或重复跳转 |

## 六、接口需求

| 接口 | 触发时机 | 作用 |
|---|---|---|
| GET /api/reward/center/detail | 进入 P6 时 | 获取分享进度、点亮进度、已领取奖励、985 礼盒资格状态、商品推荐、跳转动作 |

| POST /api/product/recommend/click | 点击「去看看」时，可选 | 记录商品点击并返回商品详情页跳转参数 |
| POST /api/tracking/event | 页面曝光、点击、异常 | 记录埋点事件 |

### 6.2 奖励展示组装规则

| 数据来源 | 展示规则 |
|---|---|
| claimed_rewards | 只放已领取的优惠券 / 折扣券，用于表示真实领取记录 |
| display_rewards | P6 实际展示列表，最多 5 张普通奖励 + 985 礼盒；未领取普通奖励 `status = unclaimed`、按钮「未领取」 |
| gift_reward | 不管是否达标，始终展示，并固定排在最后一个 |
| display_rewards 总数 ≤ 3 | 我的奖励只展示 1 排 |
| display_rewards 总数 4-6 | 我的奖励展示 2 排 |
| gift_reward.status = not_qualified | 按钮展示「未达标」，不可用 |
| gift_reward.status = qualified / unused | 按钮展示「去使用」，可点击 |

### 6.3 奖励状态建议

| reward.status | 含义 | 前端处理 |
|---|---|---|
| unclaimed | 未领取 | 展示「未领取」，不可使用 |
| unused | 已领取、可承接小程序领券 | 展示「去领取」，可点击 |
| used | 已使用 | 展示已使用态，不允许重复点击 |
| expired | 已失效 | 展示已失效态，不允许使用 |
| pending | 发放中 / 处理中 | 展示处理中，不允许重复点击 |
| not_qualified | 985 礼盒未达标 | 展示「未达标」，不可点击 |
| qualified | 985 礼盒已达标 | 展示「去使用」，可点击 |

## 八、异常处理

| 异常情况 | 页面处理 |
|---|---|
| 奖励中心接口失败 | 提示「奖励信息加载失败，请稍后再试」，提供重试 |
| session_id 缺失 | 提示「活动状态异常，请重新进入活动」 |
| 进度数据缺失 | 使用 0 分享、0 点亮兜底，不阻塞奖励展示 |
| share_target / light_target 缺失 | 使用默认 5 / 7 兜底，并记录异常埋点 |
| claimed_rewards 为空 | `display_rewards` 仍展示配置内普通奖励，普通奖励按钮显示「未领取」 |
| gift_reward 缺失 | 前端用未达标兜底卡展示，避免最后一位消失 |
| 券包跳转参数缺失 | 提示「暂时无法打开券包，请稍后再试」 |
| 商品推荐缺失 | 隐藏推荐区或展示后端兜底商品 |
| 商品详情跳转失败 | 提示「暂时无法打开商品，请稍后再试」 |
| 活动规则跳转失败 | 提示「活动规则暂时无法打开」 |
| 返回无历史栈 | 回到 P1 活动首页 |
| 用户重复点击 | 当前按钮 loading / 防抖，禁止重复请求 |
| 埋点失败 | 不阻断页面主流程，前端静默失败或重试 |

## 九、视觉与切图建议

| 模块 | 建议处理方式 | 说明 |
|---|---|---|
| 深红活动背景 | 可作为页面整体背景图 | 不承载动态业务 |
| 米金主内容卡 | 可作为静态容器背景 / CSS 容器 | 内部动态内容不要写死到图片 |
| 返回按钮 / 活动规则按钮 | 建议单独组件或切图 | 需要点击态 |
| 「我的考运进度」标题 | 前端文本渲染或标题图 | 建议文本渲染便于适配 |
| 已分享 0/5 人 | 必须前端文本渲染 | 0 和 5 都不要写死 |
| 1-7 天进度条 | 建议组件化实现 | 点亮状态必须动态；已点亮在原有圆点基础上标红，不额外叠加勾选图标 |
| 我的奖励标题 | 前端文本渲染或标题图 | 固定文案 |
| 券卡底图 | 可切券底图 / CSS 还原 | 金额、折扣、描述、状态必须动态；未领取券渲染但按钮为「未领取」 |
| 985 礼盒图 | 可作为固定最后一张奖励图 | 状态「未达标 / 去使用」必须动态 |
| 「去领取 / 去使用」小按钮 | 建议按钮组件 | 券包跳转与资格跳转需区分 action |
| 精选好物推荐卡 | 商品图与文案动态渲染 | 点击去具体商品详情页 |
| 底部「再抽一次」 | 按钮组件或按钮底图 + 文案 | 点击回 P1 首页 |

> 关键原则：P6 有大量动态数据，不建议整页切成一张静态图。背景、边框、装饰可静态，进度、奖励、商品和按钮状态必须组件化。
