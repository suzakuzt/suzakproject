## 二、页面目标

P2 只负责「抽签结果展示 + 解签入口 + 分享入口」，不承载复杂业务。核心目标是：

1. 让用户明确看到自己本次抽中的考运签；
2. 强化「高考好运 / 金榜题名」的仪式感；
3. 引导用户点击「问小璞」查看今日解签内容；
4. 引导用户点击「点击分享海报」进行分享传播；
5. 为后续 AI 解签、海报生成、商品承接保留扩展位置，但 P2 当前不直接展开复杂链路。

## 四、按钮动作

| 按钮 / 区域 | 点击动作 | 规则 |
|---|---|---|
| 问小璞 | 打开今日解签内容 | 当前建议用弹窗 / 浮层 / 占位页承接，不强制跳出 P2 |

## 五、状态逻辑

| 用户状态 / 页面状态 | 页面展示 | 点击动作 |
|---|---|---|
| 结果加载成功 | 展示签型、主签文、宜 / 忌、问小璞、分享海报入口 | 可正常解签 / 分享 |
| 结果加载中 | 展示加载态 | 暂不允许重复请求 |
| 结果加载失败 | 提示「签文加载失败，请稍后再试」 | 可重试 |
| 解签加载中 | 问小璞入口展示加载态或弹窗 loading | 等待返回 |
| 解签加载失败 | 提示「解签内容加载失败，请稍后再试」 | 可重试 |
| 海报生成中 | 展示海报生成中状态 | 防重复点击 |
| 海报生成失败 | 提示「海报生成失败，请稍后再试」 | 可重试 |

## 六、接口需求

| 接口 | 触发时机 | 作用 |
|---|---|---|
| POST /api/draw/execute | P1 点击抽签后、进入 P2 前 | 正式抽签：生成 draw_id、扣减机会、触发当天打卡 / 点亮，并返回结果入口 |
| GET /api/draw/result/detail | 进入 P2 时 | 获取本次抽签结果、签型、主签文、宜 / 忌 |
| GET /api/explain/detail | 点击「问小璞」进入 P4 时 | 获取当前签文对应的 AI 解签详情、商品福利和领取状态 |
| POST /api/poster/generate | 点击「点击分享海报」 | 生成本次签文分享海报 |
| POST /api/checkin/record | 抽签完成后由抽签流程触发 | 记录 1 天打卡 / 点亮；建议由 `draw/execute` 内部触发，每天最多点亮 1 天，P2 只读取结果不自行累加 |
| POST /api/tracking/event | 页面曝光、点击、异常 | 记录埋点事件 |

## 七、埋点事件

| 事件名 | 触发时机 | 关键参数 |
|---|---|---|
| result_page_view | 进入 P2 结果页 | activity_id、session_id、draw_id、sign_type、sign_level |

| share_poster_click | 点击「点击分享海报」 | session_id、draw_id |
| share_poster_generate_success | 海报生成成功 | session_id、draw_id、poster_id |
| share_poster_generate_fail | 海报生成失败 | session_id、draw_id、error_code、error_msg |
| result_page_load_error | P2 结果页加载失败 | session_id、draw_id、error_code、error_msg |

## 八、异常处理

| 异常情况 | 页面处理 |
|---|---|
| P2 结果接口失败 | 提示「签文加载失败，请稍后再试」，提供重试 |
| 结果数据缺失 | 提示「签文加载失败，请重新进入活动」 |
| 解签内容失败 | 提示「解签内容加载失败，请稍后再试」 |
| 海报生成失败 | 提示「海报生成失败，请稍后再试」 |
| 用户重复点击问小璞 | loading 中防重复请求 |
| 用户重复点击分享海报 | 海报生成中防重复请求 |

## 九、Agent 交付卡

### 9.1 H5 前端 Agent

任务：完成 P2 结果页展示、动态渲染、问小璞入口和分享海报入口。

必须实现：

1. 页面视觉按主设计图还原；
2. 顶部固定展示「六月考运签」；
3. 签型标签、主签文、宜 / 忌支持动态渲染；
4. 主按钮固定展示「问小璞」；
5. 底部固定展示「点击分享海报」；
6. 点击「问小璞」打开解签内容占位；
7. 点击「点击分享海报」打开海报分享占位；
8. 不在 P2 前端自行累加点亮天数；抽签完成后的打卡 / 点亮以后端记录为准，每天最多点亮 1 天；
9. 保留后续接接口位置，不要把业务写死；
10. 按埋点表触发页面曝光、点击、成功、失败事件。

### 9.2 后端 Agent

任务：提供 P2 所需结果详情、解签内容和海报生成能力。

必须实现：

1. 返回本次抽签结果详情；
2. 返回签型、签等级、主签文、宜 / 忌字段；
3. 根据 draw_id 返回对应解签内容；
4. 支持生成分享海报；
5. 返回埋点所需的 session_id、draw_id、sign_type、sign_level 等关键字段。

### 9.3 埋点 Agent

任务：按埋点表实现 P2 页面曝光、结果渲染、解签点击、海报点击、异常事件。

必须覆盖：

1. P2 结果页曝光；
2. P2 渲染成功；
3. 点击问小璞；
4. 解签成功 / 失败；
5. 点击分享海报；
6. 海报生成成功 / 失败；
7. P2 结果页加载失败。

### 9.4 QA Agent

任务：验证 P2 页面逻辑是否符合当前设计口径。

必须测试：

1. 从 P1 进入 P2 后正常展示结果页；
2. 签型标签正常展示；
3. 主签文支持动态替换；
4. 宜 / 忌内容支持动态替换；
5. 点击「问小璞」能进入解签流程；
6. 点击「点击分享海报」能进入海报流程；
7. 结果页加载失败时有异常提示；
8. 解签失败时有异常提示；
9. 海报生成失败时有异常提示；
10. 页面不出现未确认的奖励 / 商品 / 企微 / 再抽一次模块。
