# P2 H5 前端 Agent 任务卡

# 项目背景

项目：高考考运摇签 + 牛肉福利领取 H5 活动  
页面：P2 今日考运签结果页 / 解签入口页  
状态：基于主设计图整理，待确认后可进入 Agent 开发拆分

## P2 已整理业务口径

1. P2 为用户完成摇签后的结果展示页。
2. 顶部固定展示「六月考运签」。
3. 签型标签展示「金榜题名签 · 上上签」一类内容，建议支持动态配置。
4. 中间两列竖排主签文为页面核心结果区域，必须支持动态渲染。
5. 左侧展示「宜」，右侧展示「忌」，后面的内容必须支持动态渲染。
6. 中部固定展示「今日解签」与「点击下方按钮，查看今日解签内容」。
7. 主按钮固定为「问小璞」。
8. 底部次操作固定为「点击分享海报」。
9. 抽签完成后的打卡 / 点亮以后端记录为准，每天最多点亮 1 天，P2 不自行累加。
10. 当前页不默认出现再抽一次、奖励领取、商品卡、企微二维码等模块。

---

## 你的角色

你是本项目的 H5 前端 Agent，只负责 P2 结果页的页面还原、状态渲染、问小璞入口和分享海报入口。

## 本次任务目标

完成 P2 结果页的前端实现，让页面可以展示抽签结果，并能进入解签流程和分享海报流程。

## 实现范围

### 必须实现

1. 按主设计图还原页面视觉结构，包括：
   - 红色丝绒背景
   - 米金签牌主体
   - 顶部「六月考运签」
   - 签型标签区
   - 两列竖排主签文
   - 左右「宜 / 忌」展示区
   - 「今日解签」标题和说明文案
   - 主按钮「问小璞」
   - 底部「点击分享海报」

2. 结果内容动态渲染：
   - 签型标签
   - 签等级
   - 主签文
   - 宜 / 忌内容

3. 点击「问小璞」：
   - 调 `GET /api/explain/detail` 获取 AI 解签详情。
   - 成功后进入 P4。
   - 接口未接阶段可先保留占位。

4. 点击「点击分享海报」：
   - 调 `POST /api/poster/generate` 生成海报。
   - 可先做占位弹窗 / 占位页。

5. 页面状态：
   - 支持加载态；
   - 支持接口失败态；
   - 支持解签加载中；
   - 支持海报生成中。

6. 页面埋点：
   - 按埋点 Agent 定义触发前端埋点调用。

## 不要实现

1. 不要自行加入「再抽一次」按钮。
2. 不要自行加入优惠券、商品卡、企微二维码模块。
3. 不要私自改动主设计图文案结构。
4. 不要把解签流程做成复杂聊天页，除非后续确认。
5. 不要写死签文内容，需保留动态渲染能力。

```ts
type P2ResultState = {

  drawId: string;
  signType: string;
  signLevel: string;
  mainTextLeft: string;
  mainTextRight: string;
  yiText: string[];
  jiText: string[];

```

## 按钮动作映射

| 按钮 / 区域 | 前端动作 |
|---|---|
| 问小璞 | 调 `GET /api/explain/detail`，成功后进入 P4 |
| 点击分享海报 | 调 `POST /api/poster/generate`；接口未接时打开海报分享占位 |

## 异常处理

| 场景 | 前端处理 |
|---|---|
| 结果页接口失败 | 提示「签文加载失败，请稍后再试」，提供重试 |
| 结果数据缺失 | 提示「签文加载失败，请重新进入活动」 |
| 解签内容失败 | 提示「解签内容加载失败，请稍后再试」 |
| 海报生成失败 | 提示「海报生成失败，请稍后再试」 |

## 验收标准

1. 页面能正常展示设计稿结构。
2. 主签文支持动态渲染。
3. 宜 / 忌支持动态渲染。
4. 点击「问小璞」能进入解签入口。
5. 点击「点击分享海报」能进入海报入口。
6. 加载失败时有异常提示。
7. 页面不出现未确认模块。
