# P5 H5 前端开发提示词

## 页面定位

只实现 **P5 手机号领券弹窗 / 优惠券领取承接层**。

本页基于最新 P5 弹窗设计稿开发，用于承接 P4「领取专属福利」动作。当前业务不再是“领取成功后直接去使用”，而是必须先输入手机号并提交领券，后端保存手机号和领取记录后，H5 立即按接口 action 跳小程序券包 / 领券中心。

当前代码阶段 P5 仍可作为 P4 内浮层实现；后续如需要独立页面，再补路由和状态入口。

## 开发边界

1. 只处理 P5 手机号领券弹窗，不开发 P1、P2、P4、P6、P7、P8 的完整业务。
2. P4 点击「领取专属福利」后应先打开 P5 弹窗，不应直接调用发券接口。
3. P5 弹窗必须包含手机号输入框、优惠券展示和「去领取」按钮。
4. 用户输入手机号并点击「去领取」后，才调用 `POST /api/benefit/claim`。
5. 手机号必须保存到后端数据表；前端埋点不得上报手机号明文。
6. 小程序承接不得通过 URL 传手机号明文，应使用 `claim_token` / `claim_no` 识别领取记录。
7. 领取成功后，P6 我的奖励页才展示该券；未提交手机号、未领取成功的券不应进入已领取记录。
8. 不要把小程序跳转目标写死在前端；以接口返回的 `action.type` 和 `action.target` 为准。

## 已确认业务规则

1. P5 是手机号领券弹窗，不再只是领取成功页。
2. P5 主标题展示「领取优惠券」。
3. 输入框占位文案为「请输入手机号」。
4. 主按钮展示「去领取」。
5. 前端校验手机号格式：11 位中国大陆手机号。
6. 校验失败时在弹窗内提示，不调用后端。
7. 点击「去领取」时调用 `POST /api/benefit/claim`，请求体必须包含 `mobile`。
8. 后端保存手机号，建议保存明文 `receiver_mobile` 和脱敏展示 `receiver_mobile_masked`。
9. 后端生成 `claim_no` 和 `claim_token`。
10. `claim_token` 用于小程序后续识别，不使用手机号明文做跳转参数。
11. 领取成功后接口返回券信息、领取状态、`claim_no`、`claim_token` 和小程序 action。
12. `action.target` 建议包含 `claim_token` 和 `claim_no`，例如 `/pages/coupon-package/index?claim_token=xxx&claim_no=xxx`。
13. 如果后续商城可以按手机号定向发券，活动后端通过手机号或会员映射调用商城接口；当前阶段可先保存领取记录和待发券状态。
14. P5 优惠券在 `POST /api/draw/execute` 创建 draw 时固定；同一 `draw_id` 反复打开 P5 不换券，重新抽签生成新 `draw_id` 才可能换券。
15. P4/P5 前端不得在打开弹窗时重新随机券，也不得用本地图片池覆盖后端返回的 `reward.imageUrl`。

## 前端交互要求

1. P4 点击「领取专属福利」：
   - 打开 P5 弹窗；
   - 触发 `benefit_mobile_popup_view`；
   - 不立即写入领取成功记录。
2. P5 输入手机号：
   - 输入框限制数字输入；
   - 最大长度 11；
   - 支持清空和重新输入；
   - 不在页面上展示完整手机号到其他区域。
3. 点击「去领取」：
   - 校验手机号；
   - 校验通过后按钮进入 loading / 禁用；
   - 调用 `POST /api/benefit/claim`；
   - 成功后保存领取结果并按后端 action 承接；
   - 失败后恢复可点击并显示错误。
4. 领取成功后：
   - P4 领取状态更新为 `claimed`；
   - P6 我的奖励页后续能读取到该领取记录；
   - H5 自动按后端 `action` 跳小程序券包页 / 领券中心，不停留成功弹窗。

## 接口建议

### POST /api/benefit/claim

用于手机号提交领券。

请求体：

```json
{
  "session_token": "string",
  "draw_id": 123,
  "reward_code": "coupon_10",
  "mobile": "13812345678",
  "claim_channel": "h5"
}
```

成功响应：

```json
{
  "activity_id": "gaokao_lucky_sign_2026",
  "session_id": "string",
  "draw_id": 123,
  "benefit_id": "default-benefit",
  "claim_id": "123",
  "claim_no": "CL202605210001",
  "claim_token": "ct_xxx",
  "claim_status": "claimed",
  "receiver_mobile_masked": "138****5678",
  "coupon_issue_status": "pending",
  "reward": {
    "reward_type": "coupon",
    "coupon_id": "coupon_10",
    "coupon_label": "优惠券",
    "threshold_text": "无门槛",
    "amount": 10,
    "amount_text": "10",
    "unit_text": "元券",
    "coupon_name": "无门槛10元券",
    "coupon_status": "unused"
  },
  "action": {
    "button_text": "去领取",
    "type": "mini_program_coupon_package",
    "target": "/pages/coupon-package/index?claim_token=ct_xxx&claim_no=CL202605210001"
  }
}
```

### GET /api/benefit/claim/result

用于根据 `claim_no` 查询领取结果，返回字段同上。后续也可增加小程序专用查询接口：

`GET /api/benefit/claim/resolve?claim_token=xxx`

小程序通过 `claim_token` 查询领取记录、手机号、券配置和发券状态，避免 URL 携带手机号明文。

## 埋点要求

| 事件名 | 触发时机 | 关键参数 |
|---|---|---|
| `benefit_mobile_popup_view` | P5 手机号弹窗曝光 | session_id、draw_id、benefit_id、reward_code |
| `benefit_mobile_input_focus` | 手机号输入框聚焦 | session_id、draw_id、benefit_id |
| `benefit_mobile_submit_click` | 点击「去领取」 | session_id、draw_id、benefit_id、reward_code、mobile_valid |
| `benefit_mobile_validate_fail` | 手机号格式校验失败 | session_id、draw_id、benefit_id、error_code |
| `exclusive_benefit_claim_success` | 领券成功 | session_id、draw_id、benefit_id、claim_no、claim_status |
| `exclusive_benefit_claim_fail` | 领券失败 | session_id、draw_id、benefit_id、error_code、error_msg |
| `benefit_claim_result_render_success` | 领取成功状态渲染完成 | session_id、benefit_id、claim_no、reward_type、coupon_id |
| `coupon_card_exposure` | 优惠券卡曝光 | session_id、benefit_id、claim_no、coupon_id、coupon_name |
| `use_benefit_click` | 点击或自动触发小程序领券承接 | session_id、benefit_id、claim_no、coupon_id、action_type、action_target |

埋点禁止上传手机号明文，可上传 `receiver_mobile_masked` 或 `has_mobile: true`。

## 异常处理

| 场景 | 处理 |
|---|---|
| 手机号为空 | 提示「请输入手机号」 |
| 手机号格式错误 | 提示「请输入正确手机号」 |
| 重复点击去领取 | loading 中禁用按钮 |
| 接口返回已领取 | 展示已领取状态，继续使用原 `claim_no` / `claim_token` |
| 接口失败 | 提示「领取失败，请稍后重试」 |
| action 缺失 | 领取成功但提示「领券入口暂未开放」 |
| 小程序跳转失败 | 提示「暂时无法跳转，请稍后再试」 |
